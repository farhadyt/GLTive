1. Sənədin məqsədi

Bu sənəd aşağıdakıları dəqiq müəyyən edir:

Module 01 üçün əsas entity-lər
hər entity üzrə bütün field-lər
field type-ları
required/optional status
default dəyərlər
validation qaydaları
unique məhdudiyyətlər
foreign key əlaqələri
index tövsiyələri
soft delete davranışı
audit davranışı
gələcək modullara açıq qalacaq linkage nöqtələri

Bu sənəd DB truth document rolunu oynayır.

2. Ümumi model prinsipləri
2.1. Company scope

Bütün əsas business entity-lər company scope ilə işləməlidir.

2.2. Soft delete

Master entity-lərdə hard delete istifadə olunmamalıdır.

2.3. Immutable movement history

Stock movement cədvəli dəyişdirilməməli və silinməməlidir.

2.4. Serialized və quantity-based ayrılığı

Serialized item-lər ayrıca serial unit səviyyəsində izlənməlidir.

2.5. Audit readiness

Bütün write əməliyyatlar audit ilə müşayiət olunmalıdır.

2.6. Localization readiness

DB-də istifadəçiyə görünən statik etiketlər hardcoded localization ilə yox, key-based UI ilə həll olunmalıdır. DB yalnız business data saxlamalıdır.

3. Shared technical conventions

Aşağıdakı konvensiyalar bütün entity-lərdə tətbiq olunmalıdır.

3.1. ID tipi
uuid
primary key
3.2. Timestamp field-lər

Əksər entity-lərdə:

created_at → datetime with timezone
updated_at → datetime with timezone
3.3. User tracking

Uyğun olduqda:

created_by_id → FK to user
updated_by_id → FK to user
3.4. Company ownership

Business entity-lərdə:

company_id → FK to company
3.5. Active status

Master entity-lərdə:

is_active → boolean, default true
3.6. Soft delete field-ləri

Master entity-lərdə:

is_deleted → boolean, default false
deleted_at → datetime nullable
deleted_by_id → FK nullable
3.7. Notes

Optional human notes üçün:

notes → text nullable

8. Validation rules summary
8.1. Common validations
bütün string-lər trim olunmalıdır

company scope FK-lər company consistency yoxlamasından keçməlidir
8.2. Numeric validations
quantity və threshold mənfi ola bilməz
unit_cost mənfi ola bilməz
reserved > on_hand ola bilməz
8.3. Tracking validations
serialized stock_item üçün stock_serial_units işləməlidir
quantity_based item üçün serial unit yaratmaq olmaz
8.4. Warehouse validations
inactive/deleted warehouse üçün yeni movement olmaz
transfer eyni warehouse-a aparılmamalıdır
8.5. Status validations
confirmed adjustment session dəyişdirilə bilməz
immutable movement update/delete edilə bilməz
