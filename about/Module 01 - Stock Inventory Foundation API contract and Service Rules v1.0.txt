1. Sənədin məqsədi

Bu sənəd aşağıdakıları müəyyən edir:

Module 01 üçün backend API sərhədləri
endpoint qrupları

audit trigger nöqtələri

Bu sənədin rolu:
Antigravity üçün backend davranış müqaviləsi olmaqdır.

2. API dizayn prinsipləri
2.1. Company-scoped API

Bütün endpoint-lər company context ilə işləməlidir.

2.2. RESTful, amma use-case aware

Sadə CRUD olan yerlərdə REST prinsipi qorunsun, amma stok hərəkətləri kimi use-case-lər ayrıca command endpoint-lərlə verilsin.

2.3. Immutable history

Movement endpoint-ləri create-only davranışla qurulmalıdır. Update/delete endpoint verilməməlidir.

2.4. Serialized və quantity-based logic ayrıdır

Request-lərdə bu iki məntiq qarışdırılmamalıdır.

2.5. Backend enforcement məcburidir

Permission və validation yalnız frontend-də yox, backend-də də tam enforce olunmalıdır.

2.6. Audit by default

Bütün kritik write əməliyyatları audit trigger etməlidir.

2.7. Localization-ready error structure

Error code-lar sabit olsun, UI lokalizasiya etsin.

3. API base structure
Base prefix

/api/v1/stock/

Response format — success
{
  "success": true,
  "message": "optional_message_key",
  "data": {},
  "meta": {}
}
Response format — error
{
  "success": false,
  "error": {
    "code": "stock_item_not_found",
    "message": "localized_on_frontend_or_backend_key",
    "details": {},
    "field_errors": {}
  }
}
Pagination format
{
  "success": true,
  "data": {
    "items": [],
    "pagination": {
      "page": 1,
      "page_size": 20,
      "total_items": 140,
      "total_pages": 7
    }
  }
}

26. Concurrency və data integrity qaydaları

Bu modulda concurrency kritikdir.

26.1. Stock updates

Receive/issue/transfer/adjustment confirm əməliyyatlarında:

DB transaction
row-level locking və ya equivalent safe update strategy
